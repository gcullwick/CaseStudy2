Goats Galore

Some non-profit organisations do not have the budget for regular work to their websites. This can lead to a lot of these sites looking very outdated and even untrustworthy which can have a massive impact on the donations they can receive through the site.

As a group of 4 (George Cullwick, Justine Carter, Tom Bates and Jasleen Sura), we have found a site in such a situation and are going to give it a redesign to bring it up to date and increase its income.

Research

Competitor Analysis

SWOT analysis was carried out on 3 direct, and 2 indirect competitors.

User Interviews

A total of 5 interviews were carried out in a semi-strutctured format, each with 10 questions and lasted around 10 minutes.

UI Analysis

Complete analysis of the current site UI to see what the strengths and weaknesses were.

Affinity Diagram

Interview responses were sorted, grouped and categorised to produce an affinity diagram that could be used during iteration.

Feature Prioritisation Matrix

Features mentioned during the interview process were added to a prioritisation matrix to show how users would value each feature compared to how difficult they would be to impliment.

Storyboard

A storyboard was created to help visualise the steps that would be taken when using the site to help identify what needed fixing in the new design.

The Problem

During user research, the main points raised were that the website seems untrustworthy due to the UI and general design, and the lack of a donation button. This means they would be unlikely to donate.

We believe that altering the UI and creating a donation button would cause more users to be more likely to make donations as the site would seem more trustworthy and the donation process would be easier.

One way we could do this would be to redesign the UI in a more modern, complete fashion and add a donation button that draws attention and follows the user down the page.

The statement above is the problem being taken forward to solve in the ideation stage of the project.

Iteration

User Flow

We created a user flow to give the process the user would go through when using the app to decide which parts needed making for the prototype.

Mood Board & Style Guide

A mood board was produced to inspire us when developing the new site and its style. This was used to create a Style Guide to ensure the site had a cohesive style when being developed as part of a team.

Crazy 8's

The "Crazy Eights" method was used to generate some fresh starting points and get our ideas out. We could then take what we liked from these initial ideas to start designing.

Lo-Fi Prototype & Testing

Starting with a mobile first approach, we created our initial low-fidelity prototype and conducted some guerrilla user testing to look for any issues we could avoid from the start.

Hi-Fi Prototype, A/B & User Testing

We moved onto a high fidelity prototype, building upon the feedback recieved in the previous testing and then used A/B testing to make some of the design decisions.

Final Prototype

We implimented the changes from the A/B and user testing into the prototype to produce our final iteration the short time frame we had available to us for the project.
